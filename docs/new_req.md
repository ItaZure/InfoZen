## 03-16 学习日志功能升级

### 需求描述
对学习日志模块进行完整重构，支持 AI 自动生成摘要、热力图活跃度展示、日志编辑与删除。

---

### 功能详情

#### 学习日志面板

**热力图（固定显示在面板顶部）**
- 展示过去 52 周每天的提问次数，样式参考 GitHub 贡献图
- 格子颜色深浅对应当天提问次数（0/1-2/3-5/6-10/11+）
- hover 显示 tooltip：「N 次提问 · YYYY-MM-DD」
- 统计范围：该话题下所有用户消息，含软删除记录
- 数据来源：后端 `GET /api/topics/:topicId` 返回的 `activityDates` 字段

**智能日志更新按钮**
- 有未总结节点时，按钮右上角显示红色数字角标（统计对话树里未总结的节点数量，仅作提示用）
- 无未总结节点时按钮置灰，不可点击
- 点击后禁用按钮，显示「生成中...」，完成后恢复

**日志生成逻辑**
- 后端查询数据库中该话题下未总结（`summarized = 0`）且未删除（`deleted_at IS NULL`）的节点
- 按 AI 回复时间的自然日（`YYYY-MM-DD`）分组
- 各日期并行调用 DeepSeek（`deepseek-chat`）生成摘要
- 请求构造：将该日所有节点的用户+AI对话内容拼成上下文，结尾追加总结指令
  - 有旧日志：「我们之前已经做过了一些对话，你需要总结刚刚我和你的对话记录，返回给我一个有序列表，非常简要地总结我们的对话内容。你的回复只需要有序列表即可」
  - 无旧日志：「你需要总结刚刚我和你的对话记录，返回给我一个有序列表，非常简要地总结我们的对话内容。你的回复只需要有序列表即可」
- 已有旧日志时：将旧摘要内容也传给 AI，让 AI 综合后重新总结，替换旧日志
- 生成后对应节点标记为 `summarized = 1`（面包屑中显示绿色边框）
- 后端使用 upsert（`ON CONFLICT(topic_id, time_range)`）写入，幂等安全

**日志列表**
- 日志卡片按 `time_range`（`YYYY-MM-DD`）倒序排列
- 标题显示日期，内容显示 AI 生成的有序列表摘要
- hover 卡片时显示「编辑」「删除」按钮
- 编辑：inline textarea，保存/取消（TODO：接入后端更新接口）
- 删除：二次确认遮罩，确认后删除（TODO：接入后端删除接口）

**模型配置**
- 日志生成固定使用 `deepseek-chat`，配置在后端 `.env`
- 对话功能固定使用 `Gemini-3.1-Pro (Poe)`
- 暂不做模型配置页，后续按需扩展

---

### 新增后端接口

**POST /api/topics/:topicId/logs/generate**
- 前端只传话题 ID，后端自行查询该话题下未总结（`summarized = 0`）且未删除（`deleted_at IS NULL`）的节点
- 并行调用 DeepSeek API 生成各日期摘要
- upsert 写入 `logs` 表，标记 `tree_nodes.summarized = 1`
- 返回 `{ success: true, results: [{ day, summary }] }`

**PUT /api/topics/:topicId/logs/:timeRange**
- 手动编辑日志内容
- 请求体：`{ summary: string }`
- 更新 `logs` 表对应记录的 `summary`
- 返回 `{ success: true }`

**DELETE /api/topics/:topicId/logs/:timeRange**
- 软删除指定日期的日志
- 写入 `deleted_at`
- 返回 `{ success: true }`

**GET /api/topics/:topicId（新增字段）**
- 新增 `activityDates: [{ day, count }]`，统计所有用户消息（含软删除）按日期的提问次数

---

### 表结构变更

**logs 表新增唯一约束**
- `UNIQUE(topic_id, time_range)`，支持 upsert
- 兼容已有数据库：启动时重建表迁移

---

### 边界情况
- 并发点击「更新日志」：按钮禁用防重复提交
- DeepSeek 调用失败：alert 提示「日志生成失败，请稍后重试」
- 无未总结节点：alert 提示，不发起 AI 请求

## 03-15 14:30

### 需求描述
对 InfoZen AI 学习工具当前已实现功能的完整梳理。当前目标为先实现"对谈"页面的功能，支持多话题分类对话、树状上下文管理、图片输入、笔记记录和学习日志生成。

---

### 功能详情

#### 1. 全局导航（Header）
- 顶部固定导航栏，品牌名 "InfoZen"
- 三个导航入口：对谈（`/`）、拾慧（`/wisdom`）、笔记（`/notes`）
- 当前页高亮（accent 色 + 下划线）

#### 2. 对谈页（Chat）—— 核心功能

**2.1 话题切换**
- 固定五个话题：自由主题、产品技术、哲学、商业、英语
- 自由主题排在最左边，默认打开
- 每个话题维护独立的消息列表、对话树、笔记、学习日志
- 切换话题时清空当前输入框图片，保留各话题数据

**2.2 消息发送**
- 文本输入框，Enter 发送，Shift+Enter 换行
- 发送时携带：话题 ID、消息文本、图片列表、父节点 ID、上下文消息链、thinking level、web search 开关
- 发送中禁用发送按钮，显示"等待中..."
- 流式接收 AI 回复（SSE），实时追加到消息气泡
- 网络失败时显示兜底文案："抱歉，AI 服务暂时不可用，请稍后重试。"

**2.3 图片输入**
- 支持从剪贴板粘贴图片（Cmd+V）
- 最多同时上传 5 张图片，超出时 alert 提示
- 发送前在输入框上方显示缩略图预览，可逐张删除
- 图片以 base64 data URL 格式随消息一起发送
- 消息气泡内展示已发送图片缩略图，点击可全屏查看

**2.4 消息气泡**
- 用户消息：右对齐，accent 背景
- AI 消息：左对齐，card 背景，支持 Markdown 渲染（含 GFM）
- Markdown 支持：段落、标题 h1-h3、有序/无序列表、代码块（含一键复制）、行内代码、加粗、斜体、引用、链接（新标签页打开）、分割线、表格
- AI 消息底部显示时间戳（格式 YYYY-MM-DD HH:MM）
- AI 消息 hover 时显示三个操作按钮：刷新、删除、复制
- 点击消息气泡可在右侧面板同步高亮对应的对话树节点
- 新消息到达时自动滚动到底部
- MessageErrorBoundary：Markdown 渲染异常时降级为纯文本展示

**2.5 重新生成（Refresh）**
- 针对任意 AI 消息，重新向后端发起请求
- 自动还原该消息所在节点的父路径作为上下文
- 清空原 AI 消息内容后流式填充新回复

**2.6 删除消息**
- 删除某条 AI 消息时，同时删除其对应用户消息及所有子孙节点的消息
- 删除前弹出 ConfirmDialog 二次确认
- 删除后选中状态回退到父节点（无父节点则清空）

**2.7 清除对话记录**
- 清除当前话题的全部消息和对话树
- 弹出 ConfirmDialog 二次确认
- 清除时保留该话题的笔记和学习日志

**2.8 对话树 / Breadcrumb（右侧上半区）**
- 每次发送消息生成一个树节点，label 取消息前 20 字
- 新消息默认挂在当前选中节点下（作为子节点）；无选中时挂在根
- 右侧面板上半部分显示当前选中节点的路径（面包屑）
- 路径超过 4 个节点时智能省略，显示首节点 + `...` + 最近 3 个，点击 `...` 展开全部
- 点击路径中任意节点可跳转（高亮对应 AI 消息 + 滚动定位）
- 当前节点若有多个子分支，在面包屑下方展示"可选分支"列表，支持点击切换
- 已总结节点显示绿色边框，未总结节点显示默认边框
- 底部显示当前话题累计对话次数

**2.9 学习日志（右侧上半区，tab 切换）**
- 右侧面板支持"对话路径"和"学习日志"两种视图切换（logViewMode）
- 点击"更新日志"按钮，将所有未标记 summarized 的节点按小时分组，生成日志条目（格式："该时间段进行了 N 次对话"）
- 同一小时段已有日志时累加计数
- 日志按时间倒序排列
- 生成后对应节点标记为 summarized（面包屑中显示绿色边框）
- 无未总结节点时 alert 提示

**2.10 笔记（右侧下半区）**
- 使用 ReactQuill 富文本编辑器
- 工具栏支持：标题（H1-H3）、加粗、斜体、下划线、删除线、有序/无序列表、引用、代码块、文字颜色、背景色、链接、图片插入、清除格式
- 笔记内容按话题独立存储，切换话题时保留各自内容
- 清除对话记录时笔记不受影响

**2.11 AI 参数设置**
- 输入框右侧齿轮图标，点击展开设置面板（点击外部自动收起）
- Thinking Level：Low / High 两档
- Web Search：开 / 关
- 参数随每次发送请求一并传给后端

**2.12 图片全屏查看（ImageModal）**
- 点击消息中的图片缩略图，弹出全屏遮罩查看原图
- 点击遮罩或关闭按钮退出

#### 3. 后端 API（`/api/chat/send`）
- 单一 POST 接口，接收消息、图片、上下文、话题 ID、thinking level、web search
- 按话题加载对应 system prompt（从 `server/prompts/` 目录读取 .md 文件）
- 调用 Poe API（模型：Gemini-3.1-Pro），透传 SSE 流式响应给前端
- 重复内容检测：web search 场景下若新 chunk 开头与已积累内容重合则提前终止流
- 图片以 `image_url` 类型拼入 user message content
- 请求体 JSON 上限 50mb（支持 base64 图片）

#### 4. System Prompt 配置
- 四个话题各有独立 prompt 文件：产品技术.md、哲学.md、商业.md、英语.md
- 服务启动时一次性加载，加载失败降级为空 prompt

#### 5. 页面路由
- `/`：对谈（已实现）
- `/wisdom`：拾慧（占位页，Coming Soon）
- `/notes`：笔记（占位页，Coming Soon）

3. **学习日志内容过于简单**：当前日志仅记录"该时间段进行了 N 次对话"，暂不接入 AI 自动生成摘要。
